Generation and evaluation of Polish notation

Polish notation (prefix) is a mathematical notation in which operators precede their operands, in contrast to the more common infix notation, in which operators are placed between operands.

Why polish notation?

The idea is simply to have a parenthesis-free notation that makes each equation shorter and easier to parse in terms of defining the evaluation priority of the operators.

Converting infix to prefix.

Converting infix to prefix is done by making use of two stacks, one for the operators and one for the operands.

  1. For every term in expression
    1. If it’s an operand
      1. Push to evaluation stack.
    1. If it’s an operator
      1. While the stack is not empty and the last operator’s precedence is equal or higher than the term
        1. Pop the operator from the operator stack
        1. Compute the operation between the last two operands in the evaluation stack
        1. Push the result back in the evaluation stack
      1. Push operator to operator stack
    1. If it’s “(”
      1. Push to operator stack
    1. If it’s “)”
      1. While the last operator in the stack is not equal to “(”
        1. Pop the operator from the operator stack
        1. Compute the operation between the last two operands in the evaluation stack
        1. Push the result back in the evaluation stack
      1. Pop the operand from the stack
  1. While the operand is not empty
    1. Pop the operator from the operator stack
    1. Compute the operation between the last two operands in the evaluation stack
    1. Push the result back in the evaluation stack
  1. Evaluation contains only one element, being the expression in prefix form.

Converting prefix to infix

Converting prefix to infix is done quite easily, since there are no paratheses. Although the end result is quite messy.

  1. For every token in expression
    1. If the token is an operator
      1. Pop two elements
      1. Push operation of the two elements between paratheses
    1. If the token is negation
      1. Pop one element and add negation
      1. Push to stack
    1. If the token is an operand
      1. Push to stack

def isoperator(token):
    if token in ['+', '-', '*', '/', '^', '%', '&', '|', '-', '=']:
        return True
    else:
        return False

def polishtoinfix(exp):

Computing prefix

Takes the same steps as converting to infix, but calculates the operation between elements.

The usable operations supported by this program are:

  • + (addition)
  • - (subtraction) and - as a unary function
  • * (multiplication)
  • / (division)
  • % (modulo)
  • ^ (power) using existing python function **

def polishcalculator(exp):
    expression = [x for x in exp.split()]
    expression = expression[::-1]
    stack = []
    for token in expression:
        match token:
            case "+":
                aux = stack.pop() + stack.pop()
                stack.append(aux)
            case "-":
                aux = stack.pop() - stack.pop()
                stack.append(aux)
            case "*":
                aux = stack.pop() * stack.pop()
                stack.append(aux)
            case "/":
                aux = stack.pop() / stack.pop()
                stack.append(float("{:.2f}".format(aux)))
            case "%":
                aux = stack.pop() % stack.pop()
                stack.append(float("{:.2f}".format(aux)))
            case "^":
                a = stack.pop()
                b = stack.pop()
                if a < 0:
                    aux = -a ** b
                else:
                    aux = a ** b
                stack.append(float("{:.2f}".format(aux)))
            case _:
                stack.append(int(token))

def neg(a):
    if a == 1:
        return 0
    return 1

def lor(a,b):
    if a == 0 and b == 0:
        return 0
    return 1

def land(a ,b):
    if a or b:
        return 1
    return 0

def lto(a ,b):
    if a == 0 and b:
        return 0
    return 1

def equal(a,b):
    if a == b:
        return 1
    return 0

def polishlogicalcalculator(exp):
    expression = [x for x in exp]
    expression = expression[::-1]
    stack = []
    final = []
    for token in expression:
        match token:
            case "!":
                a = stack.pop()
                stack.append(neg(a))
            case "|":
                a = lor(stack.pop(), stack.pop())
                stack.append(a)
            case "&":
                a = land(stack.pop(), stack.pop())
                stack.append(a)
            case "-":
                a = lto(stack.pop(), stack.pop())
                stack.append(a)
            case "=":
                a = equal(stack.pop(), stack.pop())
                stack.append(a)
            case _:
                stack.append(int(token)
